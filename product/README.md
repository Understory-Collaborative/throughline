# Product

This directory holds how Throughline should be built. It is the shared
reference for everyone touching the product, including engineers, designers,
writers, and community collaborators.

If you are about to change copy, a question in a flow, or a screen, read the
relevant doc here first.

## What lives here

- `pillars/` the principles that decide what we build and how we build it
- `personas/` who we build for, written with and validated by the people we
  serve
- `design-system/` voice, tone, accessibility, and the visual language
- `first-step-out/` the brief and content model for the first feature

## A note on how this gets written

We collaborate with formerly incarcerated individuals in design and building.
Drafts in this directory that have not yet been reviewed with that community
are marked **Draft, needs community review**. Treat those as starting points
for a conversation, not as settled fact.

We are building a tool, not a community. A digital tool does not replace a
community of people. Software can support the work. It cannot deliver it. Keep
that in view when a doc here suggests the app should do something a person
should do.
